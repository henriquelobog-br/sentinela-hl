# Documento 112.5 — Persistência PostgreSQL/Supabase

Persiste o resultado do pipeline no schema existente. **Nenhuma migration nova.**
psycopg direto (sem ORM), queries parametrizadas, uma transação por item com
rollback automático.

## Princípio

**Nenhuma regra científica mora no writer.** Ele não decide nada — traduz o que o
`DecisionEngine` já decidiu em linhas. O mapeamento é uma **tabela**, não lógica:

| decisão | `pipeline_status` | cria evento | `requires_human_review` |
|---|---|---|---|
| `pass` | `validated` | sim | `false` |
| `escalate` | `escalated` | sim | `true` |
| `reject` | `rejected` | **não** | — |

`reject` **preserva claim + classifications** como trilha de auditoria — só não
promove a evento.

## Mapeamento das tabelas

| Tabela | Conteúdo |
|---|---|
| `raw.sources` | upsert da fonte (idempotente por name+kind) |
| `raw.fetch_runs` | uma linha por execução da coleta |
| `raw.items` | o `RawItem` coletado |
| `knowledge.claims` | claim provisória do Evidence Builder |
| `knowledge.classifications` | os três julgamentos do Agente 113 |
| `knowledge.events` | apenas `pass` e `escalate` |

**`run_id`** vai em `classifications.detail->>'run_id'` — evita migration
prematura. Se o painel precisar consultar direto, aí sim vira coluna indexada.

## API

```python
class PostgresWriter:
    ensure_source(source) -> UUID            # idempotente
    start_fetch_run(source_id) -> UUID
    finish_fetch_run(id, status=, items_*=)
    find_duplicate(source_id, item) -> UUID | None
    persist_item(item, source_id, fetch_run_id) -> (UUID, is_new)
    persist_result(raw_item_id, filter_input, filter_output) -> dict
```

Alto nível — **tudo em uma transação**:

```python
persist_pipeline_result(conn, raw_item, filter_input, filter_output, source, fetch_run_id)
```

Se o `FilterAgentOutput` vier com `ok=false` (erro do agente), o item bruto é
registrado e **nada entra no conhecimento** — o erro não vira dado.

## Deduplicação

1. `(source_id, external_id)` quando houver `external_id`;
2. `content_hash` como fallback.

Item duplicado devolve o `id` existente com `is_new=False`, sem reinserir.
