# Documento 112.2B — Pipeline Collector → Builder → Agente

Fecha a Milestone 2 na parte de coleta. Substitui os fixtures do 112.1 por uma
entrada real: `Collector → Builder`. Não reimplementa nada — só costura os
contratos já congelados.

> Item-a-item. Sem batch, async, fila ou paralelismo (Milestones futuras).

## Fluxo

```
Collector → RawItem → ClaimBuilder → FilterAgentInput → run_filters → FilterAgentOutput
```

Cada peça é intocada: o pipeline chama os contratos, não os altera.

## Arquivo

```
sentinela/pipeline/
├── run_pipeline.py
└── tests/test_pipeline.py
```

## Dois modos (o mesmo fluxo, duas fronteiras)

`run_pipeline.py` expõe funções puras e um CLI com dois modos:

| Modo | Faz | Para quê |
|---|---|---|
| `emit-input` | Collector → Builder → imprime `FilterAgentInput` (JSONL) no stdout | o **n8n** consome: `run_pipeline ... \| run_filters -` |
| `run` | Collector → Builder → **Agente** (`evaluate`) → `FilterAgentOutput` | pipeline Python completo, testável com FakeLLMClient |

Por que dois modos: `run` é o caminho Python puro (rápido, determinístico nos
testes). `emit-input` preserva a fronteira stdin/stdout congelada — o n8n
continua chamando `run_filters -` como caixa-preta, agora alimentado pelo
collector em vez do `cat fixture`.

## Como o n8n muda (Execute Command)

Só troca o lado esquerdo do pipe. O `run_filters -` e o Switch não mudam:

```bash
# 112.1 (fixture)
cat sentinela/filters/tests/fixtures/caso_pass.json | uv run python -m sentinela.cli.run_filters -

# 112.2B (coleta real, item-a-item)
uv run python -m sentinela.pipeline.run_pipeline --collector rss --url <FEED> --mode emit-input \
  | head -1 | uv run python -m sentinela.cli.run_filters -
```

Cada linha do `emit-input` é um `FilterAgentInput`. O contrato stdin/stdout
congelado garante que o agente nem percebe a troca da fonte.

## Item-a-item

O `emit-input` emite um `FilterAgentInput` por linha (JSONL). O `run_filters`
processa **um** por vez, exatamente como no 112.1. Batch/async/fila ficam para
depois — hoje o volume não justifica.

## Como testar

```bash
uv run pytest sentinela/pipeline/tests/
```

Tudo offline (FakeLLMClient). Cobre: `build_inputs` produz `FilterAgentInput`,
pipeline completo com mock e com rss (XML local), e a invariante item-a-item
(N RawItem → N FilterAgentOutput).

## Robustez

O modo `emit-input` trata `BrokenPipeError` — se o consumidor fechar o pipe cedo
(ex.: `head`), encerra em silêncio em vez de cuspir traceback.
