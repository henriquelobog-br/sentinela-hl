# Documento 112.1 — Estrutura do Workflow n8n (esqueleto)

Primeira etapa da Milestone 2. Objetivo: validar o **esqueleto** do workflow —
n8n chamando o agente 113 como caixa-preta e roteando pelo veredito — antes de
plugar coleta real ou Supabase.

> Escopo: mockado. Sem RSS real. Sem Supabase. Sem regra de negócio no n8n.

## Fluxo

```
Schedule Trigger (cron)
   → Coleta mockada (serve um fixture como FilterAgentInput)
   → Execute Command (run_filters -, via stdin)   ← caixa-preta
   → Parse JSON (lê o FilterReport)
   → Switch (roteia por report.decision)
        ├── pass     → log
        ├── escalate → log
        └── reject   → log
```

Os dois nós que importam são o `Execute Command` e o `Switch`. O resto é
encanamento. Tudo o que não for esses dois deve permanecer trivial.

## Nós

### 1. Schedule Trigger
Cron diário. No esqueleto, dispara manualmente para teste.

### 2. Coleta mockada
Não coleta nada. Entrega um objeto que **já é** um `FilterAgentInput` válido —
reutiliza os fixtures existentes (`caso_pass`, `caso_escalate`, `caso_reject`).
Não inventa formato: serve o que o `run_filters` já espera.

### 3. Execute Command — o coração do princípio "n8n burro"
Chama o agente exatamente como no terminal e trata a saída como JSON opaco.

**Contrato de entrada/saída (congelado):**

| | |
|---|---|
| Comando | `python -m sentinela.cli.run_filters -` |
| Entrada | `FilterAgentInput` em JSON, via **stdin** |
| Saída (stdout) | `FilterAgentOutput` em JSON |
| Exit code | `0` = pass/escalate · `1` = reject ou erro |

O n8n envia o JSON por stdin:

```bash
echo '{...FilterAgentInput...}' | uv run python -m sentinela.cli.run_filters -
```

O n8n **não** sabe o que é proveniência, rótulo ou calibração. Executa e recebe.

### 4. Parse JSON
Converte o stdout do comando em objeto. Lê o campo `report.decision`.

### 5. Switch — a única "decisão" do n8n (que não é decisão de negócio)
Roteia por `report.decision` — um valor de string que o **`DecisionEngine` já
calculou**. O n8n reage ao veredito pronto; não decide se algo passa.

```
report.decision == "pass"     → branch pass
report.decision == "escalate" → branch escalate
report.decision == "reject"   → branch reject
```

**Linha vermelha:** no dia em que aparecer um nó de `IF` perguntando
`confidence_score > 0.7`, a lógica vazou para o n8n e o princípio quebrou.

### 6. Log (3 saídas)
Cada branch só registra o resultado (no esqueleto, um nó de log/NoOp). Persistência
em Supabase é o Documento 112.5, não aqui.

## Formato da entrada (`FilterAgentInput`)

```json
{
  "claim":   { "raw_item_id": "...", "statement": "...",
               "epistemic_status": "hypothesis", "confidence_score": 0.62 },
  "source":  { "excerpt": "...", "url": "...", "reliability": "high" },
  "context": { "case_id": "...", "scope": "science_research" }
}
```

## Formato da saída (`FilterAgentOutput`)

```json
{
  "ok": true,
  "report": {
    "decision": "escalate",
    "requires_human_review": true,
    "classifications": [ { "filter": "...", "result": "...", "rationale": "..." } ]
  },
  "run_id": "..."
}
```

Em falha: `{ "ok": false, "error": { "error": "llm_error | invalid_json | ...", "message": "..." } }`.
O agente **nunca** crasha nem chuta veredito — sempre um envelope estruturado.

## Alteração no CLI (incluída neste documento)

`run_filters.py` passou a aceitar `-` como fonte, lendo o JSON de **stdin**,
mantendo compatibilidade com caminho de arquivo:

```bash
python -m sentinela.cli.run_filters caminho.json      # arquivo (compat)
echo 'JSON' | python -m sentinela.cli.run_filters -   # stdin (n8n)
cat input.json | python -m sentinela.cli.run_filters -
```

Robustez adicionada: se faltarem as variáveis do OpenRouter, o CLI retorna um
`llm_error` estruturado em stdout (exit 1), em vez de crashar.

## Critérios de aceite do esqueleto

- [ ] O workflow dispara pelo Schedule Trigger.
- [ ] A coleta mockada entrega um `FilterAgentInput` válido (um fixture).
- [ ] O Execute Command chama `run_filters -` e recebe o JSON em stdout.
- [ ] O Switch roteia corretamente para `pass`, `escalate` e `reject`.
- [ ] Rodando os três fixtures, os três branches são exercitados.
- [ ] Nenhum nó do n8n contém regra de negócio (só `Switch` por `decision`).
- [ ] Nada é gravado no Supabase; nenhuma fonte RSS real é lida.
