# Documento 112.4 — Scientific Validation Report

Responde a pergunta que persiste desde o 113: **o pipeline produz decisões boas
com dados reais?** Roda `RSS real → Collector → Builder → OpenRouter real →
Agente` e materializa uma ficha revisável para o Henrique.

> Sem banco, sem n8n, sem Supabase. Só executar o pipeline e reportar.

## Saída

```
reports/validation-YYYY-MM-DD.json   registro completo (fonte da verdade)
reports/validation-YYYY-MM-DD.html   ficha legível (derivada do json)
```

O JSON tem tudo (o `FilterAgentOutput` de cada item + sinais + timing). O HTML é
a leitura humana — dark, indigo, Inter, denso, zero motion (segue o kit).

## Cada item vira uma ficha

- **statement** + fonte com link + badge da decisão (`pass`/`escalate`/`reject`)
- **sinais do builder**: reliability, category, confidence, rótulo provisório, keywords
- **excerpt da fonte** (o texto que a proveniência auditou)
- **auditoria dos filtros**: cada filtro com resultado + **o rationale do LLM**
- **timing e tokens** por item + `run_id`

## Arquivos

```
sentinela/validation/
├── collect_report.py   # roda o pipeline, monta o dict do report
├── render_html.py      # dict -> HTML (kit de design)
├── run.py              # CLI
└── tests/test_validation.py
```

## Como rodar (dados reais — na sua máquina)

```bash
export OPENROUTER_API_KEY=...
export OPENROUTER_MODEL_PRIMARY=...
uv run python -m sentinela.validation.run --collector rss --url <FEED_CIENTIFICO>
```

Feeds sugeridos (dentro do escopo): Nature, NASA, NOAA, Copernicus, USGS.

## Instrumentação

Tempo por item e da chamada LLM. Tokens vêm do campo `usage` do OpenRouter
(capturado em `OpenRouterClient.last_usage`, sem quebrar o contrato). **Custo em
$ não é reportado** — os tokens ficam no relatório e você multiplica pelo preço
do seu modelo. Chutar preço aqui seria impreciso.

## O que testar na prática (Teste A e B do plano)

- **A — RSS real:** os sinais do builder fazem sentido? reliability bate com o
  domínio? category acerta? statement é limpo? keywords são relevantes?
- **B — OpenRouter real:** a decisão é boa? o rationale de cada filtro é
  coerente? o tempo e o custo por item são aceitáveis?

Se as respostas forem "sim", o Supabase (112.5) passa a armazenar dado confiável.
