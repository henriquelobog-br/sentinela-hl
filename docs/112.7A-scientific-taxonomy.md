# Documento 112.7A — Scientific Taxonomy

Primeira camada da arquitetura de priorização científica (ADR-007). Um
**vocabulário científico controlado** — não uma lista de palavras-chave.

> Determinístico. Sem LLM. Sem embeddings. Sem banco. Sem chamadas externas.

## O que é (e o que não é)

A taxonomia representa **conceitos científicos**, não notícias. Cada conceito
tem identidade estável (`id`), sinônimos, um pai opcional (hierarquia) e
**relações** com outros conceitos. A relação importa mais que o sinônimo:

```
Aerosols
├── Mineral Dust ──related──▶ Atmospheric Transport, Trade Winds, Iron Oxides
│   ├── Saharan Dust ──related──▶ Amazon Basin, South Atlantic
│   └── Namib Dust ──related──▶ South Atlantic, Southeast Brazil, Southern Africa
├── Volcanic Ash ──related──▶ Volcanic Eruption
└── Smoke
```

É isso que permitirá às camadas seguintes conectar um texto sobre "aeolian dust
in the South Atlantic" à linha de pesquisa Namíbia→Espírito Santo, mesmo sem
nenhuma menção literal a "Namíbia".

A taxonomia **não conhece pesquisadores** (isso é o Research Profile, 112.7B) e
**não faz matching nem scoring** (Interest Engine, 112.7E).

## Estrutura

```
sentinela/taxonomy/
├── models.py      # Concept / Domain / Taxonomy (Pydantic v2) + normalize_term
├── loader.py      # YAML por domínio → Taxonomy, erros claros com arquivo
├── validator.py   # integridade referencial (lista TODOS os problemas)
└── taxonomy.py    # TaxonomyIndex: lookup O(1), hierarquia, vizinhança

taxonomy/          # dados (1 YAML = 1 domínio científico)
├── astronomy.yaml          ├── remote_sensing.yaml
├── atmospheric.yaml        ├── seismology.yaml
├── climate.yaml            ├── space_weather.yaml
├── geology.yaml            ├── volcanology.yaml
├── geopolitics.yaml        └── regions.yaml   (vocabulário geográfico compartilhado)
└── oceanography.yaml
```

Semente atual: **11 domínios · 57 conceitos · 240 termos · 94 relações.**

## Garantias do validator

- ids de conceito **globalmente únicos** (entre domínios);
- `parent` e `related` referenciam conceitos existentes;
- sem auto-relação; sem **ciclo** na hierarquia;
- `domain` do conceito = domínio do arquivo onde vive;
- **nenhum termo ambíguo**: um sinônimo não pode mapear para dois conceitos;
- erros listados todos de uma vez, com nome de arquivo no caso do loader.

## API de consulta (`TaxonomyIndex`)

```python
tax = load_taxonomy("taxonomy")
idx = TaxonomyIndex(tax)

idx.find_by_term("AEOLIAN DUST")   # → Concept(mineral_dust)  · case/acento-insensível
idx.ancestors("namib_dust")        # → [mineral_dust, aerosols]
idx.descendants("aerosols")        # → toda a subárvore
idx.related("mineral_dust")        # → vizinhança 1 salto (simétrica na consulta)
idx.concepts_of_domain("geology")
idx.stats()
```

## Expansão

Adicionar conceito = editar o YAML do domínio (ou criar um novo arquivo). O
loader valida tudo no carregamento; conceito com referência quebrada não passa.
A estrutura permite evolução gradual para um Knowledge Graph (ADR-007, 112.9) —
**não implementado aqui**, deliberadamente.

## Nota honesta sobre o conteúdo

Os 57 conceitos são uma **semente correta**, com atenção especial à linha de
pesquisa da poeira (Namíbia→ES: mineral dust, alísios, 850 hPa, MERRA-2,
CALIPSO, Atlântico Sul). Taxonomia científica é curadoria contínua do
pesquisador — o formato YAML por domínio existe para o Henrique expandir sem
tocar em código.

## Critérios de aceite (validados)

- [x] Pydantic v2; ids snake_case; sinônimos dedupli­cados case-insensitive
- [x] YAML inválido falha com mensagem clara (arquivo + problema)
- [x] integridade referencial completa (parent, related, ciclos, ambiguidade)
- [x] lookup por termo é case-insensitive e insensível a acento
- [x] hierarquia (ancestors/descendants) e vizinhança (related simétrico)
- [x] caso científico real testado: "aeolian dust" → mineral_dust → transporte/alísios; namib_dust conecta southern_africa ↔ south_atlantic ↔ southeast_brazil
